# Changelog Generator

gitコミット履歴から、ユーザーフレンドリーなチェンジログを作成する。

## Use when

- 新バージョンのリリースノート準備
- 週次・月次のプロダクトアップデートまとめ
- 顧客向け変更ドキュメント作成
- PR作成時の変更サマリ

## 処理フロー

1. **Git履歴スキャン**: 指定期間またはバージョン間のコミットを分析
2. **変更分類**: 機能追加、改善、バグ修正、破壊的変更、セキュリティに分類
3. **技術→ユーザー言語変換**: 開発者向けコミットを顧客向け文章に変換
4. **ノイズ除去**: 内部コミット（リファクタ、テスト、chore等）を除外
5. **フォーマット整形**: 構造化されたチェンジログを生成

## 使い方

### 基本

```bash
# 直近リリースからの変更
前回リリースからのチェンジログを作成して

# 直近1週間の変更
過去1週間のコミットからチェンジログを作成
```

### 期間指定

```bash
# 日付範囲
3月1日から3月15日までのチェンジログを作成

# タグ間
v2.4.0からv2.5.0までのチェンジログを作成
```

## 出力フォーマット

```markdown
# アップデート - 2024年3月10日週

## 新機能

- **チーム機能追加**: プロジェクトごとにワークスペースを作成可能に
  - チームメンバーを招待して共同作業

- **キーボードショートカット**: `?`キーで利用可能なショートカット一覧を表示

## 改善

- **同期速度向上**: ファイル同期が2倍高速に
- **検索機能強化**: ファイル名だけでなく本文も検索対象に

## バグ修正

- 大きな画像がアップロードできない問題を修正
- 予約投稿のタイムゾーン表示を修正
- 通知バッジのカウント表示を修正

## 技術的な変更

- Next.js 15.1.0へアップデート
- 依存関係のセキュリティアップデート
```

## カテゴリマッピング

| コミットtype | チェンジログカテゴリ |
|-------------|-------------------|
| feat | 新機能 |
| fix | バグ修正 |
| perf | 改善 |
| security | セキュリティ |
| BREAKING CHANGE | 破壊的変更 |
| refactor, test, chore, docs, style, ci | 除外（内部変更） |

## Tips

- プロジェクトのルートディレクトリから実行
- `CHANGELOG.md`に直接追記する場合は確認してから
- 公開前に内容をレビュー
- スコープ（ec/backend）で変更箇所を明確に
