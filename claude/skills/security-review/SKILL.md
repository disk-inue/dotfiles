# Security Review Skill

## Use when

- コードレビューでセキュリティチェックを依頼されたとき
- 新機能実装後のセキュリティ確認
- 認証・認可に関わるコード変更時
- 外部入力を扱うコードの実装時
- APIエンドポイントの追加・変更時

## チェック項目

### 1. インジェクション対策（OWASP A03:2021）

- **SQLインジェクション**: Prismaのパラメータ化クエリを使用しているか
- **NoSQLインジェクション**: 動的クエリ構築を避けているか
- **コマンドインジェクション**: シェルコマンド実行を避けているか
- **XSS**: ユーザー入力のサニタイズ、React/Next.jsのエスケープ機能を利用しているか

### 2. 認証・認可（OWASP A01:2021, A07:2021）

- **認証**: next-auth/Firebase Authの適切な実装
- **認可**: GraphQL Shieldによるアクセス制御
- **セッション管理**: JWTの適切な有効期限設定
- **パスワード**: 適切なハッシュ化（使用している場合）

### 3. 機密データ保護（OWASP A02:2021）

- **環境変数**: シークレットがハードコードされていないか
- **ログ出力**: 機密情報がログに含まれていないか
- **レスポンス**: 不要な情報が返されていないか
- **エラーメッセージ**: スタックトレースや内部情報の漏洩がないか

### 4. セキュリティ設定（OWASP A05:2021）

- **CORS**: 適切なオリジン設定
- **CSP**: Content Security Policyヘッダー
- **Rate Limiting**: APIエンドポイントの保護
- **HTTPS**: 本番環境でのTLS強制

### 5. コンポーネントの脆弱性（OWASP A06:2021）

- **依存関係**: 既知の脆弱性を持つパッケージがないか
- **バージョン**: サポート終了したバージョンを使用していないか

## 出力フォーマット

```markdown
## セキュリティレビュー結果

### 重大な問題
- [問題の説明と該当箇所]
- [修正方法]

### 警告
- [潜在的な問題]
- [推奨される対策]

### 確認済み
- [問題なしと判断した項目]
```
