# Debug Skill

バグの原因を特定し、修正方法を提案する。

## Use when

- エラーが発生しているとき
- バグの原因を調査するとき
- Sentryエラーを調査するとき
- パフォーマンス問題を調査するとき

## デバッグ手順

### 1. 症状の把握

- **エラーメッセージ**: 完全なエラーメッセージとスタックトレース
- **再現手順**: どうすれば再現できるか
- **環境情報**: 開発/本番、ブラウザ、Node.jsバージョン
- **発生タイミング**: いつから発生しているか

### 2. 情報収集

```bash
# ログ確認
# Sentryダッシュボード確認
# ブラウザコンソール確認

# 関連コードの確認
grep -r "エラーに関連するキーワード" apps/

# 最近の変更確認
git log --oneline -20
git diff HEAD~10 -- <関連ファイル>
```

### 3. 仮説立案

原因の候補をリストアップ：

1. **コード変更**: 最近のコミットで壊れた可能性
2. **依存関係**: パッケージ更新による互換性問題
3. **環境差異**: 開発と本番の設定差異
4. **データ**: 想定外のデータ入力
5. **外部サービス**: API障害、レート制限

### 4. 検証

```bash
# git bisectで原因コミット特定
git bisect start
git bisect bad HEAD
git bisect good <動作していたコミット>

# 最小再現ケース作成
# 仮説ごとに検証
```

### 5. 修正と確認

- 修正コードを実装
- テストで再発防止を確認
- 関連箇所への影響確認

## Sentryエラー調査

### フレームワーク/ライブラリ問題の確認

```bash
# Next.js関連の既知問題を検索
gh search issues "エラーメッセージ" --repo vercel/next.js

# React関連
gh search issues "エラーメッセージ" --repo facebook/react

# その他ライブラリ
gh search issues "エラーメッセージ" --repo <owner>/<repo>
```

### エラー分類

| 分類 | 対応 |
|-----|-----|
| フレームワーク既知問題 | ignoreErrorsに追加検討 |
| アプリケーションコード | 即時修正 |
| ブラウザ拡張機能 | ignoreErrorsに追加 |
| ネットワークエラー | リトライ実装検討 |

## よくあるパターン

### Next.js App Router

- Server/Client Component境界の問題
- use clientの不足
- Server Actionsのエラー
- キャッシュ問題

### GraphQL

- N+1問題
- 認証エラー
- スキーマ不一致

### Prisma

- トランザクション失敗
- 接続プール枯渇
- マイグレーション不整合

### React

- hydrationエラー
- useEffect依存配列の問題
- Stateの不正更新
